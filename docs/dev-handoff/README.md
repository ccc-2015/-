# 开发交接索引

这个目录用于保存继续开发所需的固定上下文。后续重要功能完成后，应同步更新这里，避免只依赖聊天记录。

## 文档

- `current-status.md`：当前已完成能力、未完成能力、关键提交。
- `development-workflow.md`：日常开发、验证、提交和推送流程。
- `runbook.md`：本地启动、默认账号、常用检查命令。
- `recovery.md`：开发中断、服务异常、Git 状态异常时的恢复方式。
- `next-steps.md`：建议继续开发路线和验收点。
